# Reproducibility Runbook

This runbook defines the non-visual research workflow for FibroDynMix. It is
intended for manuscript review, package maintenance, and handoff to another
analyst.

## Lightweight Quality Gate

Run this gate after code, documentation, or manifest changes:

```bash
Rscript scripts/run_project_maturity_audit.R
Rscript scripts/run_reproducibility_audit.R
Rscript scripts/run_runtime_lock.R
Rscript scripts/check_project_integrity.R
Rscript -e 'testthat::test_local()'
R CMD build .
R CMD check --no-manual FibroDynMix_0.0.0.9000.tar.gz
```

Remove generated check artifacts after validation:

```bash
rm -rf FibroDynMix.Rcheck FibroDynMix_0.0.0.9000.tar.gz Rplots.pdf
```

## Analysis Map

The generated analysis map is stored under
`analysis/reproducibility_audit/`:

- `analysis_catalog.tsv` links each analysis to its run script, output
  directory, primary manifest, research role, and claim boundary.
- `script_inventory.tsv` lists scripts, approximate scope, and whether they
  write analysis outputs.
- `doc_inventory.tsv` lists documentation files and whether they include claim
  boundaries or output references.
- `reproducibility_audit_manifest.tsv` summarizes whether all registered
  analyses have output directories and primary manifests.

## Runtime Lock

The package-level runtime lock is stored under `analysis/runtime_lock/` and is
generated by:

```bash
Rscript scripts/run_runtime_lock.R
```

It records the R version, key `Suggests` package versions, external public data
DOI/GEO accessions, local data files, and heavy-analysis expected inputs and
outputs. The reader-facing description is `docs/package-runtime-lock.md`.

## Claim Discipline

The reproducibility audit verifies that evidence files exist and that each
analysis has a stated boundary. It does not prove that every biological
interpretation is true. For manuscript language, prefer these bounded claims:

- raw-count negative-binomial modeling rather than gene-set scoring;
- latent simplex state estimation rather than direct weighted scores;
- simulation-calibrated uncertainty diagnostics rather than full all-parameter
  Bayesian inference;
- public count-matrix validation rather than FASTQ-level atlas reprocessing;
- downstream representation utility rather than universal downstream
  superiority;
- NB outer-iteration convergence sensitivity rather than assuming `n_outer=2`
  is optimizer convergence;
- core higher-outer NB reruns as optimizer diagnostics, not proof that training
  NB objective selects the best latent-state recovery;
- cross-sectional transition-flow summaries rather than lineage tracing;
- HCC GSE246215 results as QC-sensitive and hypothesis-generating.
